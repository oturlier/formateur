image:vscode1.png[150,150,role="related thumb right"] {uri_vscode}[{vscode}, window="_blank"] est un *éditeur de texte* est bourré de fonctionalités dépassant de loin l'édition de ressources au formats asciidoc! +
Son gros *avantage* réside dans la possibilité d'avoir un *panneau de pré-visualisation* permettant de se donner une idée de ce à quoi cela pourrait ressembler une fois le fichier `.adoc` compilé vers un fichier `.html` ou `.pdf`. +

WARNING: {vscode} est addictif... on y passe des heures... Vous aurez étés prévenus!


== Installation

.Procédure : installation de {vscode}
. se rendre sur {uri_vscode}[le site web de {vscode}, window="_blank"] (clic sur le lien précédent ou sur l'image suivante pour ouvrir un autre onglet).
+
image::vscode2.png[link="{uri_vscode}", window="_blank"]
. cliquez sur *Download for Windows* pour télécharger la dernière version adaptée
. double-cliquez pour installer en suivant les instructions

== Bonnes pratiques / Configuration

Ci-dessous quelques infos utiles pour éditer des ressources au format {asc} avec {vscode}

=== Configuration initiale

.Francisation (installation de plugin depuis {vscode-market} )
* effectuer la recherche sur google avec ces mots clés `francisation vscode`
* on tombe sur la page de {vscode-market} nous proposant d'installer le _French Language Pack for Visual Studio Code_
* yapluka cliquer et suivre les directives ...
* en redémarrant {vscode} il devrait être en français

.Asciidoctor (installation de plugin depuis {vscode})
* kbd:[F1] ouvre la "palette" de commande
* tapez `ext install asciidoctor-vscode`
* on peut aussi installer un `plugin` depuis internet, le {uri_marketplace-vscode}[Market place {vscode}, window="_blank"] . Voici ce qu'une recherche avec le terme `asciidoc` donne (image à cliquer) :  image:vscode3.png[link="{uri_marketplace-vscode-ascidoc}", window="_blank"]

=== Bonnes pratiques

.Mise en forme du texte "`englobante`"
-> idem {subl} : _voir le tuto_  xref:inst+conf_subl.adoc#mise-en-forme-englobante[Mise en forme englobante]

.Sélection multiple
* idem {subl} : sélection texte -> kbd:[Ctrl+D] successif -> édition simultanée des zones sélectionnées

.Partition d'écran
* en *3 parties* (de la gauche vers la droite) : +
panneau *explorateur* -> panneau *éditeur* -> panneau *prévisualisation*
* voir l'image ci-dessous (image à cliquer pour zoom)
+
.Aperçu de {vscode} éditant la page actuelle
image::vscode4.png[link="_images/vscode4.png"]


.Config panneau prévisualisation pour être "blanc" lorsque le thème de {vscode} est sombre
[options="compact"]
* affichage panneau latéral *extensions* kbd:[Ctrl+Maj+X] ou icône "cubes"
* aller à *Asciidoc* & clic sur la petite roue de config en bas à droite image:vscode5.png[role="related thumb right"]
* un onglet dans {vscode} s'ouvre :
** cliquer sur l'onglet *Utilisateur* en haut (réglages pour l'utilisateur)
** dans la liste des réglages, descendre à `Asciidoc:Preview>Use Editor Style` & décocher cette option
** faire de même dans l'onglet *Espace de travail*


[TIP]
====
.{vscode} n'est pas évident à configurer car ses capacités sont très vastes (à l'images d'un logiciel de DAO "objet" BIM...) :

* difficultés pour installer le correcteur orthographique Français ...
* édition des fichiers de configuration _vs_ install clic/clic/clic
* nombreux _plugins_ (+ de 10 000...), tous plus intéressants que les autres... mais risquant d'affecter le bon fonctionnement en cas d'incompatibilité  (préocupations identiques dans l'écosystème BIM...)
* _pour autant_, il semble être *le plus abouti des éditeurs* pour ce que l'on veut en faire (édition de fichiers texte au format `.adoc`)

.Stratégie édition ressources :

* configuration de {dblcdr} avec {notpad} comme éditeur par défaut (léger, rapide à ouvrir avec le raccourci kbd:[F4], garde en mémoire tous les fichiers ouverts même si nn enregistrés)
* travail avec {subl} pour commencer `.adoc`
* portage progressif de l'édition sur {vscode}
* compilation avec *antora* dans une fenêtre de commande (raccourci pour l'ouverture dans le dossier : kbd:[F9]) pour inclusion de cette ressource sur un site web ...
====
